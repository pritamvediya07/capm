# CAPM trial execution flow — does the design-doc defence actually work?

This is the **sequenced run plan** for one trial that confirms (or refutes) the
defence specified in `Cross_Agent_Provenance_Defense_Design.docx`. Run style:
**Efficient** — cache on, each chain built once per content and scored by all 5
defenses, single model (`gemini-2.5-flash`). Most phases are crypto/scoring and
cost **zero** requests; the model-backed headline costs a handful.

> **Measured quota reality:** `gemini-2.5-flash` free tier =
> **20 requests/DAY** (`GenerateRequestsPerDayPerProjectPerModel-FreeTier`), not
> the 250 we first assumed. The responder paces under the per-minute limit,
> caches every call (re-runs are free), and — when the daily 20 is exhausted —
> **degrades to a deterministic paraphrase** rather than crashing. This is sound
> because CAPM's verdict is computed from the signed manifest, not the relay
> text, so the containment result (ASR) is identical whether a hop's content was
> Gemini-generated or the deterministic stand-in.

The flow is ordered so each phase answers one question the design doc raises,
and so the cheap/foundational evidence is in hand before any quota is spent.

## What "the defence works" means (decision criteria)

The design doc's falsifiable hypotheses (§6.3) and claims (§7). The trial
**confirms the defence** iff:

| ID | Hypothesis / claim | Pass criterion | Experiment |
|---|---|---|---|
| H1 / C2 | Provenance survives N cross-org hops where baselines carry none | CAPM prov-survival = 1.00; identity/flat structured chain = 0 | E1.1, s2 |
| H2 / C1 | CAPM down-weights laundering all baselines accept | CAPM ASR ≪ baselines, statistically (McNemar p<0.05) | E1.1 |
| H3 / C5 | Warrant monotone non-increasing; strictly lower for low-warrant origin | monotone curve holds; attacks capped | s2, E2.2 |
| H4 / C5 | Verification overhead sub-ms per hop | per-hop verify < 1 ms | E6.1 |
| C3 | Adversarial cross-org provenance + text-only recovery | tamper/forgery → REJECT; soft-binding recovers | E2.3, s3 |
| C4 | Sound under the Semantic-Laundering theorem (external warrant) | warrant decided outside the model; forgery infeasible | E2.1, E3.3 |
| — (honesty) | The honest boundary is real, not hidden | origin-capture accepted **but attributable** | E3.2 |

If H1–H4 pass with **real Gemini content** and the adaptive adversaries (E3.x)
do not break containment, the defence works for this testbed. E3.2 is the honest
limit we report rather than hide.

## The sequence

### Phase 0 — Mechanism sanity (0 requests)
Confirm the wiring before spending quota.
1. `python -m tests.test_capm` — 13 unit tests.
2. `python -m experiments.s0_single_hop_honest` … `s3_textonly_and_tamper` — the S0–S3 ladder.
3. `python -m experiments.run_all` — deterministic headline table.
4. `PYTHONPATH=vendor/saga CAPM_USE_SAGA=1 python -m experiments.validate_against_saga` — runs on SAGA's real crypto.

**Gate:** all pass → proceed. (This is the mechanism; Phase 1 is the realism.)

### Phase 1 — Real-model containment — the core question (~23 requests)
Does CAPM contain laundering when the relayed content is generated by a *real*
model, not a deterministic stub?
5. `python -m experiments.e4_1_real_responders --llm` — Gemini relay faithfulness (~3 req).
6. `python -m experiments.e1_1_main_matrix --llm` — the **headline**: ASR + Wilson CIs + McNemar vs every baseline, with Gemini content, efficient (~20 req).

**Gate (H1+H2+C1+C2):** CAPM ASR ≈ 0 with CI excluding the baselines' ≈1.0, McNemar p<0.05, prov-survival 1.00. This is the make-or-break result.

### Phase 2 — Adaptive adversary — does it hold when the attacker knows CAPM? (0 req)
7. `python -m experiments.e3_3_manifest_forgery` — forgery → REJECT (add `CAPM_USE_SAGA=1` for real crypto).
8. `python -m experiments.e3_4_collusion` — warrant origin-bounded regardless of #colluders.
9. `python -m experiments.e3_1_lying_transformation` — VERBATIM-lie detected.

**Gate (C4):** no adaptive adversary raises warrant or evades rejection.

### Phase 3 — The honest boundary — where the defence stops (0 req)
10. `python -m experiments.e3_2_origin_capture` — class-lie accepted but attributable.

**Gate (honesty):** confirms the headline ASR is a real mechanism, not an
artifact; documents origin-integrity as a separate composable layer.

### Phase 4 — Soundness & necessity — why it works (0 req)
11. `python -m experiments.e2_1_soundness` — ProVerif model (binding unforgeable).
12. `python -m experiments.e2_3_forgery_battery` — 10-case tamper battery, none accepted.
13. `python -m experiments.e8_ablations` — each removed component raises ASR / trades soundness.

**Gate (C3+C4):** binding sound; every component shown load-bearing.

### Phase 5 — Deployability & trade-off (0 req)
14. `python -m experiments.e6_1_overhead_scaling` — per-hop verify sub-ms (H4).
15. `python -m experiments.e7_1_frontier` — utility–resistance Pareto curve.
16. `python -m experiments.e7_2_false_positive` — over-blocking on honest workloads.

**Gate (C5):** sub-ms overhead; a dominating operating point exists.

### Phase 6 — Extended realism (optional, budget permitting; ~50 req)
Run only if Phases 1–5 pass and quota remains:
17. real E5.1/E5.2/E5.3 (corpus stand-ins → real), E1.2 lossy survival, E4.3 source bias.
